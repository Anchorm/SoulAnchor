How to modify SoulAnchor

Resources can be found in $XDG_DATA_DIRS/soulanchor

  • /usr/share/soulanchor
  • $HOME/.local/share/soulanchor
  • $HOME/.config/soulanchor/soulanchor.conf
  1. database
    • a. translation
    • b. bookname language
    • c. dictionary
  2. audio bible
  3. music
  4. notes
  5. color schemes
  6. gui language
  7. code

1. Database

1.a translations

To REMOVE a translation:

  • delete the table, e.g. t_kjv
  • remove the kjv row in the bible_version_key table

To ADD a translation:

Since version 1.09 translations can be imported from within SoulAnchor (menu view-import).

Manually it can be done this way:

Add a table with the name t_translation
It should have these columns for book number / chapter number / verse number and verse text:

  • INTEGER b
  • INTEGER c
  • INTEGER v
  • TEXT t

Add a row in the bible_version_key table. Content should be self-explanatory. The abbreviation and version column are used for the combo boxes.

I can recommend the excellent DB Browser for SQLite: https://sqlitebrowser.org

1.b booknames language

If you want to add booknames in your language, add a table in the books-i18n.db database. Content should be self-explanatory.

1.c add/remove a dictionary in dictionaries.db

  • add a table, e.g. newdict, with topic and definition text columns
  • add a row in the info table, with strong field = false

The strong = true rows are used for the bottom part of the info window

I used dictionaries that are also used in the android mybible app (https://mybible.zone)

2. audio bible

If you have mp3's of bible CHAPTERS you can place them in the audio-bible directory (~/.local/share/soulanchor/audio-bible).
The filename should begin with 2 digits, for the bookname detection. And it should end with the chapter indication, 3 digits, and .mp3.

E.g.

  • 01 005.mp3
  • 01_005.mp3
  • 01005.mp3
  • 01 Genesis 005.mp3

Now the "play this chapter" right click context menu option can be used (only when one chapter is shown).

3. music

If you have Biblically inspired mp3's you can place them in the music folder (~/.local/share/soulanchor/music).

They will be added to the contemplations menu on startup. If the filename has a scripture reference then SoulAnchor should jump to the book and chapter. (except Hymns, those are checked for a lyrics file and shown in the info frame)

Psalms: place in the music/psalms folder and start the filename with "psalm" or "Psalm", and a number. E.g. Psalm 1 blessed is the man.mp3

Hymns: place in the music/hymns folder and if there is a .txt file with the same name, it should be displayed in the info frame. E.g. Come, Thou Long-Expected Jesus.mp3 + Come, Thou Long-Expected Jesus.txt

Other inspired music: can be placed in the music folder. E.g. 1-cor-13-love.mp3
Valid scripture links are:

  • book chapter
  • book chapter:verse
  • book chapter:verse-verse

4. notes

Create a note file (~/.local/share/soulanchor/notes)

In plaintext or in markdown format, or html. All .txt and .md files in the notes directory will be added to the text contemplations menu.

Links can be created like this:

  • <a href="bible:heb 6:19">Heb. 6:19</a>
  • <a href="bible:ps 1-2">Psalm 1-2"</a>
  • <a href="strongs:G1">G1</a>

5. color schemes

Can be defined in soulanchor.conf in ~/.config/soulanchor
in html color format under: [Schemes]

For example:

amber=#775F18,#3E2A11,#b29a00,#fff7bf#ffee80,#5BF4DA
(name)(number)(text)(title)(background 1)(background 2)(highlight/clash)

6. gui language

The soulanchor_en.ts in the data/locales source folder can be used with the Qt Linguist application in order to produce a soulanchor_yourlanguagecode.qm file. The .qm file should be placed in the system data dir /usr/share/soulanchor/locales/

The 'yourlanguagecode' part of the filename will be used in the settings menu for the GUI language combobox.

7. coding

If you want to make changes in the code i recommend using Qt Creator.
